# 通用撰写规范（所有面试类型必读）

## 默认字数


| 面试官类型      | 建议字数             |
| ---------- | ---------------- |
| 专业面（业务/技术） | 500–1200 字，结构化   |
| HR 面       | 400–800 字，结论清晰   |
| 交叉面        | 500–1000 字，按比重取中 |


## 叙述原则

- 用 **STAR / 行为证据**；区分「面试中观察到的行为」与「推断」。
- 不利判断用「待验证」「建议复核」「面试中未充分展现」，避免定论式人身评价。
- 面试官的推荐理由/补充直接融入正文（亮点或待验证段落），不单独列"面试官补充"区块。

## 禁忌（所有输出必须遵守）

- **歧视**：性别、年龄、婚育、地域、民族、宗教信仰等。
- **无依据**：未经证实的背调结论、道德标签、医疗式诊断。
- **泄密**：未公开的产品 / 战略 / 客户合同 / 财务数据。
- **承诺**：对薪酬、职级、期权、编制、入职时间等作确定性承诺。
- **绝对化**：「肯定 / 一定 / 绝不」，除非有明确制度依据。
- **综合分制**：不使用 1–5 分、X/5、百分比匹配度作为录用结论。

## 措辞示例

- 宜：「在讨论 [场景] 时，候选人描述了 [具体做法]，体现出 [能力]。」
- 忌：「此人不行」「情商低」「肯定干不久」「综合匹配度 58%」。
