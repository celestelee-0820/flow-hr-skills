# 面试评价记录生成

## Persona 读取（每次调用优先执行）

在任何步骤开始前，先读取用户 persona 中已存储的面评偏好，直接沿用，无需重新询问。

Persona 结构化字段：

```
[固定偏好]
- 原文展示：是（默认）/ 否（用户明确关闭后记录）

[背景信息]
- 面试官所在行业：从历史输入推断后记录
- 面试官岗位：从历史交互中推断记录

[用户强调的关注点]
- 用户特别强调的评估维度或标准（如「主动性对我很重要」）
- 后续面评中对应维度优先呈现

[外部上传的定制内容]
- 用户上传的偏好文档、团队评估标准、自定义模板等的结构化摘要

[不存储]
- JD、候选人信息（每次面试不同，不跨场景保留）
```

Persona 被动积累规则（不主动问，按以下触发条件记录）：
- 用户说「不要原文」→ 记录原文展示 = 否
- 成功从输入推断出行业 → 记录面试官所在行业
- 用户本次强调某个关注点 → 追加到关注点列表
- 用户上传偏好文档 → 结构化摘要后存入

---

## Step 0：输入采集

**首要输入**（自动获取，优先级最高）：
- `meeting_transcript` — 面试对话记录（获取方式待补充）
- `meeting_meta_data` — 会议元数据，含会议时间、参会人列表（获取方式待补充）

**辅助输入**（非必须，有则用）：
- 岗位 JD — 重要辅助，可显著提升面评质量
- 候选人简历 — 用于补充候选人背景和基本信息

**路由参数**（从输入中推断，推断不出则询问）：

| 参数 | 推断来源 | 推断失败时 |
|------|---------|-----------|
| 候选人层级 | JD / 职位名称 / transcript | 不确定则询问 |
| 面试官类型 | 用户提供 / meeting_meta_data | 无则默认专业面 |
| 面试阶段 | 轮次信息 / transcript | 不确定则询问 |
| 行业 | JD / 简历 / transcript（任一来源均可）| 无法推断则留空，不询问 |

**面试阶段逻辑：**
- 首轮：独立生成，不依赖前序记录
- 后续轮：搜索同名候选人既往记录；有则在面评开头附「前序面试参考」（1–3 句），无则跳过

---

## Step 1：事前提问（收集面试官主观输入）

向面试官提出以下问题，**已有的输入直接跳过对应问题**：

> 1. **面试中有哪些让你印象深刻的点？有哪些不满意的部分？**（必问）
>
> 2. 是否有岗位 JD？提供后可显著提升面评质量。（已有 JD 则跳过）
>
> 3. 是否有候选人简历？（已有简历则跳过）

收到回答后进入 Step 2。

---

## Step 2：加载文件 & 提取行为证据

**加载规则（严格按需，不多读）：**

| 条件 | 加载文件 |
|------|---------|
| 必读 | `dimensions/junior.md` 或 `senior.md` 或 `manager.md`（三选一） |
| 必读 | `templates/professional.md` 或 `templates/hr.md`（二选一；交叉面合并两者关键维度） |
| 仅当行业可从任意输入中确认 | `industry-norms/<行业>.md`（含该行业附加维度和写作规范） |

> 行业无法从 JD / 简历 / transcript 中推断时，跳过 industry-norms 文件，使用通用原则。

**面试官轮次 → 考察层激活：**
- 一面 → 聚焦**深度**维度
- 二面 → 聚焦**广度**维度
- 三面 → 聚焦**高度**维度
- Manager+ 候选人 → 一面起即含广度维度

**提取原则（严格遵守）：**
- 只写候选人实际说过的内容；不推测、不补充候选人未提及的信息
- 基本信息字段只填可直接来自输入的内容，有就填，无则留空
- 有行为证据的维度才写入表格；无证据的维度直接省略
- 维度名称不附加来源标注（不写「互联网附加维度」等）
- 模板结构仅供参考，按实际面试内容取舍，不强制填满所有维度

---

## Step 3：合并输入 & 生成完整面评

按面试官类型加载的模板生成最终输出，融合规则：
- 面试官提供的印象/不满意内容 → 融入「亮点」或「待验证」段落
- 原文引用默认展示（persona 设置关闭时除外）→ 作为 blockquote 内嵌于对应维度行下方
- 不输出「总体推荐」行，除非用户明确要求

---

## 通用撰写规范

**字数：**

| 面试官类型 | 建议字数 |
|-----------|---------|
| 专业面 | 500–1200 字 |
| HR 面 | 400–800 字 |
| 交叉面 | 500–1000 字 |

**叙述原则：**
- 用 STAR / 行为证据；区分「观察到的行为」与「推断」
- 不利判断用「待验证」「建议复核」「未充分展现」，避免定论式评价

**禁忌：**
- 歧视：性别、年龄、婚育、地域、民族、宗教
- 无依据：未经证实的背调结论、道德标签、医疗式诊断
- 泄密：未公开的产品 / 战略 / 客户合同 / 财务数据
- 承诺：对薪酬、职级、期权、编制、入职时间等作确定性承诺
- 综合分制：不使用 1–5 分、百分比匹配度作为录用结论
