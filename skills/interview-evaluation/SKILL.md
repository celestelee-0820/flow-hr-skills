# 面试评价记录生成

## Step 0：输入收集 & 路由参数确认

**① 必填输入**
面试 transcript（会议记录 / 对话文本）

**② 可选输入**（有则用，无则按需询问）
岗位 JD、候选人简历、面试官 notes

**③ 路由参数**（从输入推断；推断不出则询问）

| 参数 | 取值 | 推断失败时 |
|------|------|-----------|
| 候选人层级 | Junior / Senior / Manager+ | 从 JD 或职位名称推断；不确定则问 |
| 面试官类型 | 专业面（业务/技术）/ HR 面 / 交叉面 | 用户提供；无则默认专业面 |
| 面试阶段 | 首轮 / 后续轮 | 从轮次信息推断；不确定则问 |
| 行业 | 互联网 / 消费品 / 制造业 / 金融 / 医疗 / 咨询 | 从 JD 直接确认；无法确认则跳过行业文件 |

**④ 面试阶段逻辑**
- **首轮**：独立生成面评，不依赖前序记录
- **后续轮**：搜索是否有同名候选人的既往面试记录；有则在面评开头附「前序面试参考」（1–3 句摘要），无则跳过

## Step 1：生成前提问（收集面试官主观输入）

在处理 transcript 之前，先向面试官提出以下问题（一次性列出，等待回答后再继续）：

> **生成面评前，需要你确认几个信息：**
>
> 1. **本次面试你的总体倾向是？**
>    推荐进入下一轮 / 待定 / 不推荐
>
> 2. **推荐理由（选填）**
>    如有 transcript 未能体现的判断，或对下一轮的建议，可在此补充。
>
> 3. **面评中是否需要展示对话原文？**
>    是 / 否

收到回答后进入 Step 2。

---

## Step 2：加载文件 & 提取行为证据

**加载规则（严格按需，不多读）：**

| 条件 | 加载文件 |
|------|---------|
| 必读 | `dimensions/junior.md` 或 `senior.md` 或 `manager.md`（三选一） |
| 必读 | `templates/professional.md` 或 `templates/hr.md`（二选一；交叉面合并两者关键维度） |
| 仅当行业从JD中可直接确认 | `industry-norms/<行业>.md`（包含该行业的附加维度和写作规范） |

> 行业无法从输入直接确认时，跳过 industry-norms 文件，直接使用通用原则生成面评。

**面试官轮次 → 考察层激活：**
- 一面 → 聚焦**深度**维度
- 二面 → 聚焦**广度**维度
- 三面 → 聚焦**高度**维度
- Manager+ 候选人 → 一面起即含广度维度

**提取原则（严格遵守）：**
- **只写候选人实际说过的内容**；不推测、不补充候选人未提及的信息
- **有行为证据的维度才写入表格**；无证据的维度直接省略，不写占位行
- 若某维度有价值但证据薄，在「待验证」段落简短说明，不在表格中展开
- **维度名称不附加来源标注**（不写「互联网附加维度」等）；直接写维度名
- 模板结构仅供参考，按面试实际内容取舍，不强制填满所有维度
- 原文引用段落只写维度名作标题，不写内部操作说明

**内部草稿格式：**

```
[候选人基本信息] 姓名 / 应聘职位 / 面试日期 / 面试官 / 面试阶段
[前序面试参考]（后续轮且有记录时）既往轮次核心结论 1–3 句
[分维度观察] 维度 | 档位 | 关键观察（行为证据）| 考察层
[亮点] 候选人最打动人的 2–3 个表现
[待验证] 证据薄或下一轮需核实的点
```

---

## Step 3：合并输入 & 生成完整面评

按面试官类型加载的模板生成最终输出，融合规则：
- 面试官提供了推荐理由 → 融入「亮点」或「待验证」段落，不单独列区块
- 用户选择展示原文 → 在每条维度观察后附 transcript 原文片段（直接展示，标题只写维度名）
- 文档末尾只保留一行面试官标注：`总体推荐：[推荐进入下一轮 / 待定 / 不推荐]`

## 通用撰写规范（内联，无需读取 common.md）

**字数：**

| 面试官类型 | 建议字数 |
|-----------|---------|
| 专业面 | 500–1200 字 |
| HR 面 | 400–800 字 |
| 交叉面 | 500–1000 字 |

**叙述原则：**
- 用 STAR / 行为证据；区分「观察到的行为」与「推断」
- 不利判断用「待验证」「建议复核」「面试中未充分展现」，避免定论式人身评价

**禁忌（所有输出必须遵守）：**
- 歧视：性别、年龄、婚育、地域、民族、宗教
- 无依据：未经证实的背调结论、道德标签、医疗式诊断
- 泄密：未公开的产品 / 战略 / 客户合同 / 财务数据
- 承诺：对薪酬、职级、期权、编制、入职时间等作确定性承诺
- 综合分制：不使用 1–5 分、百分比匹配度作为录用结论
