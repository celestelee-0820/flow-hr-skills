## name: interview-evaluation
description: >-
  Generate interview evaluation records from meeting transcripts, optional JD, resume, and
  interviewer notes. Routes by candidate level, interviewer type, interview stage, and industry.
  Use when asked for 面试评价, 面评, interview feedback, interview assessment, or
  post-interview evaluation documents.

# 面试评价记录生成

## Step 0：输入收集 & 路由参数确认

**① 必填输入**
面试 transcript（会议记录 / 对话文本）

**② 可选输入**（有则用，无则按需询问）
岗位 JD、候选人简历、面试官 notes

**③ 路由参数**（从输入推断；推断不出则询问）


| 参数    | 取值                             | 推断失败时                                     |
| ----- | ------------------------------ | ----------------------------------------- |
| 候选人层级 | Junior / Senior / Manager+     | 从 JD 或职位名称推断；不确定则问                        |
| 面试官类型 | 专业面（业务/技术）/ HR 面 / 交叉面         | 用户提供；无则默认专业面                              |
| 面试阶段  | 首轮 / 后续轮                       | 从轮次信息推断；不确定则问                             |
| 行业    | 互联网 / 消费品 / 制造业 / 金融 / 医疗 / 咨询 | 从 JD 推断；不确定则先用 `industry-norms/common.md` |


**④ 面试阶段逻辑**

- **首轮**：独立生成面评，不依赖前序记录
- **后续轮**：搜索是否有同名候选人的既往面试记录；有则在面评开头附"前序面试参考"（1–3 句摘要），无则跳过，不影响流程

---

## Step 1：生成前提问（收集面试官主观输入）

在处理 transcript 之前，先向面试官提出以下问题（一次性列出，等待回答后再继续）：

> **生成面评前，需要你确认几个信息：**
>
> 1. **本次面试你的总体倾向是？**
>   推荐进入下一轮 / 待定 / 不推荐
> 2. **推荐理由（选填）**
>   如有 transcript 未能体现的判断，或对下一轮的建议，可在此补充。
> 3. **面评中是否需要展示对话原文？**
>   是 / 否

收到回答后进入 Step 2。

---

## Step 2：加载文件 & 提取行为证据

**加载规则（不多读）：**


| 路由结果          | 加载文件                                                              |
| ------------- | ----------------------------------------------------------------- |
| 候选人层级         | `dimensions/junior.md` 或 `senior.md` 或 `manager.md`（三选一）          |
| 行业附加维度（若行业已知） | `dimensions/industry-extras.md`                                   |
| 合规规范（必读）      | `industry-norms/common.md`                                        |
| 行业细则（若行业已知）   | `industry-norms/<行业>.md`（六选一）                                     |
| 面试官类型         | `templates/professional.md` 或 `templates/hr.md`（二选一）；交叉面则合并两者关键维度 |


**面试官类型 → 维度激活规则：**

- 专业面 → 聚焦标注「专业面」的维度，挖技术深度与项目证据
- HR 面 → 聚焦标注「HR 面」的维度，考察动机、沟通、文化适配
- 交叉面 → 两类维度均激活，按各自权重排列

**面试官轮次 → 考察层激活规则：**

- 一面（直属主管）→ 聚焦**深度**标签维度
- 二面（总监）→ 聚焦**广度**标签维度
- 三面（GM）→ 聚焦**高度**标签维度
- Manager+ 候选人 → 一面起即含广度维度

**从 transcript 提取，形成内部草稿：**

```
[候选人基本信息] 姓名 / 应聘职位 / 面试日期 / 面试官 / 面试阶段
[前序面试参考]（后续轮且有记录时）既往轮次核心结论 1–3 句
[分维度观察] 维度 | 考察层 | 面试官类型 | 档位 | 行为证据（STAR）| 原文片段（若用户选"是"）
[亮点] 候选人最打动人的 2–3 个表现
[待验证 / 证据不足] 哪些维度未充分展现或需下一轮核实
```

---

## Step 3：合并输入 & 生成完整面评

**融合规则：**

- 面试官提供了推荐理由 → 融入面评"亮点"或"待验证点"段落，不单独列"面试官补充"区块
- 若用户选择展示对话原文 → 在每条维度观察后附对应 transcript 原文片段（直接展示原文，非引用格式）
- 若用户选择不展示 → 仅保留 AI 提炼的摘要描述

**按面试官类型加载模板，生成最终输出。**

文档末尾只保留一行面试官标注：

```
总体推荐：[推荐进入下一轮 / 待定 / 不推荐]
```

**通用原则：**

- 全文不使用综合分制（1–5 分、X/5、百分比匹配度）作为录用结论
- 有行为证据再写判断；无证据标「证据不足」
- 中文输出，除非用户要求英文
- 遵守 `industry-norms/common.md` 合规禁忌
